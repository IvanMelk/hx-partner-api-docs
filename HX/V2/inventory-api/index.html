Inventory models

There are four main datamodels in the files returned from the API. Please see the model description for each of the models below for details. These models are generated from the booking and content systems so the contents of these are dynamic. Properties described in the documentation will only be shown if there is a value for it.

File descriptions

Each file contains a FileMetadata object which shows the server creation time of the file contents. Please use this information to check whether you are using the latest available data.

Example :

"fileMetadata": {
        "creationTimeUtc": "2023-08-25T09:55:48.2582222Z"
    }

Some files are provided in a localized format, indicated by a filename like filename_[locale] or filename_[market]_[currency]. The market files you see depend on which market you, as a Hurtigruten partner, are eligible to obtain.

Package

Filename format : packages_[market]_[currency].json

This file contains an updated list of packages, available cabins and prices. This data is refreshed on a 24 hour schedule. For exact pricing, you will need to issue a quote request to the booking API.

Excursion

Filename format : excursion_[locale].json

This file contains information about excursions offered by Hurtigruten. Details about specific excursions can only be obtained after placing a quote for a package.

Voyage

Filename format: voyages_[locale].json

This file contains information about the packages listed in the Packages file. The content includes, for example:

  • Links to images
  • Selling points
  • Passenger information
  • Itinerary information
Note

Make sure to use the voyage file that corresponds to the package file you are using as you will otherwise get incomplete or incorrect content information about the packages. For example:

  • packages_uk_aud --> voyages_EN-APAC
  • packages_us_usd --> voyages_EN-AMERICAS
  • packages_no_dkk --> voyages_da-DK

Portlist

Filename format : portlist.json

This file contains a list of port codes used in the Packages and Voyages files. The port codes do not follow a specific standard so please use this file to link port code to country and location.
